POST /api/media
Crea un mitjà

Examples

POST /api/media
{
  "medium": {
    "name": "new medium",
    "path": "http://www.newmedium.com/"
  }
}
201
{
  "path": "http://www.newmedium.com/",
  "os_family": null,
  "created_at": "2016-09-30 07:54:59 UTC",
  "updated_at": "2016-09-30 07:54:59 UTC",
  "id": 1011586619,
  "name": "new medium",
  "operatingsystems": [],
  "locations": [],
  "organizations": []
}

Params

Param Name Description
medium
Required

Validations:

  • Must be a Hash
medium[name]
Required

Nom dels mitjans

Validations:

  • Must be String
medium[path]
Required

The path to the medium, can be a URL or a valid NFS server (exclusive of the architecture).

for example http://mirror.centos.org/centos/$version/os/$arch where $arch will be substituted for the host’s actual OS architecture and $version, $major and $minor will be substituted for the version of the operating system.

Solaris and Debian media may also use $release.

Validations:

  • Must be String
medium[os_family]
Optional , <span class="translation_missing" title="translation missing: ca.apipie.nil_allowed">Nil Allowed</span>

Família de sistema operatiu, valors disponibles: AIX, Altlinux, Archlinux, Coreos, Debian, Freebsd, Gentoo, Junos, NXOS, Redhat, Solaris, Suse, Windows, Xenserver

Validations:

  • Must be String
medium[operatingsystem_ids]
Optional , <span class="translation_missing" title="translation missing: ca.apipie.nil_allowed">Nil Allowed</span>

Validations:

  • Must be an array of any type
