POST /api/media
Create a medium.

Examples

POST /api/media
{
  "medium": {
    "name": "new medium",
    "path": "http://www.newmedium.com/"
  }
}
201
{
  "config_path": null,
  "created_at": "2014-07-23T13:31:55Z",
  "id": 1011586620,
  "image_path": null,
  "media_path": null,
  "name": "new medium",
  "os_family": null,
  "path": "http://www.newmedium.com/",
  "updated_at": "2014-07-23T13:31:55Z"
}

Params

Param name Description
medium
optional , nil allowed
Value: Must be a Hash
medium[name]
required

Name of media


Value: Must be String
medium[path]
required

The path to the medium, can be a URL or a valid NFS server (exclusive of the architecture).

for example http://mirror.centos.org/centos/$version/os/$arch where $arch will be substituted for the hostâ€™s actual OS architecture and $version, $major and $minor will be substituted for the version of the operating system.

Solaris and Debian media may also use $release.


Value: Must be String
medium[os_family]
optional , nil allowed

The family that the operating system belongs to.

Available families:

  • AIX
  • Altlinux
  • Archlinux
  • Debian
  • Freebsd
  • Gentoo
  • Junos
  • Redhat
  • Solaris
  • Suse
  • Windows

Value: Must be String
medium[operatingsystem_ids]
optional , nil allowed
Value: Must be Array
