katello_manifest – Manage Katello manifests¶

  • Synopsis
  • Requirements
  • Parameters
  • Examples
  • Status

Synopsis¶

  • Upload and Manage Katello manifests

Requirements¶

The below requirements are needed on the host that executes this module.

  • nailgun >= 0.29.0
  • python >= 2.6
  • ansible >= 2.3

Parameters¶

Parameter Choices/Defaults Comments
manifest_path
-
Path to the manifest zip file
This parameter will be ignored if state=absent or state=refreshed
organization
- / required
Organization that the manifest is in
password
- / required
Password for user accessing Foreman server
redhat_repository_url
-
Default:
https://cdn.redhat.com
URL to retrieve content from
server_url
- / required
URL of Foreman server
state
-
    Choices:
  • absent
  • present ←
  • refreshed
The state of the manifest
username
- / required
Username on Foreman server
verify_ssl
boolean
    Choices:
  • no
  • yes ←
Verify SSL of the Foreman server

Examples¶

- name: "Upload the RHEL developer edition manifest"
  katello_manifest:
    username: "admin"
    password: "changeme"
    server_url: "https://foreman.example.com"
    verify_ssl: false
    organization: "Default Organization"
    state: present
    manifest_path: "/tmp/manifest.zip"
